## 1. 문제

이번 과제의 목표는 머꼬(Mowkow) 문법을 사용하여 계승(factorial) 프로그램을 작성하고, 그 구현 과정과 사용한 언어 기능을 정리하는 것이다.  
계승은 어떤 자연수 `n`에 대하여 `n * (n-1) * (n-2) * ... * 1`로 정의되며, 특별히 `0! = 1`로 정의된다.  
따라서 프로그램은 입력으로 정수 `n`을 받아서, `n`의 계승값을 계산하여 출력할 수 있어야 한다.

머꼬는 Scheme 계열 문법과 유사한 우리말 리스프 언어이며, 이번 과제에서는 재귀 호출과 조건식을 이용하여 계승 함수를 구현한다.  
즉, 문제를 다시 말하면 **머꼬 문법으로 계승 함수를 정의하고, 입력값에 대해 올바른 결과를 출력하는 프로그램을 작성하는 것**이다.

---

## 2. 전략

계승 문제는 재귀(recursion)로 매우 자연스럽게 표현할 수 있다.  
수학적으로 계승은 다음과 같이 정의된다.

- `0! = 1`
- `n! = n × (n-1)!`  (`n > 0`)

따라서 프로그램에서도 먼저 **기저 사례(base case)** 와 **재귀 사례(recursive case)** 를 나누어 생각하였다.  
입력값이 0이면 결과는 1을 반환하고, 0이 아니라면 현재 값 `n`에 `n-1`의 계승을 곱하도록 하면 된다.

구현 과정은 다음과 같이 정리할 수 있다.

1. `팩토리얼`이라는 이름의 함수를 정의한다.
2. 함수는 하나의 인자 `n`을 받는다.
3. `n`이 0인지 조건식으로 검사한다.
4. 0이면 1을 반환한다.
5. 0이 아니면 `n * factorial(n-1)`을 계산한다.
6. 입력을 받아서 정의한 함수에 전달하고, 결과를 출력한다.

이 전략의 장점은 문제의 수학적 정의와 프로그램 구조가 거의 일치한다는 점이다.  
즉, 복잡한 반복문 없이도 계승의 정의를 그대로 코드로 옮길 수 있어서 구현이 비교적 직관적이었다.

---

## 3. 언어의 기능

이번 프로그램 구현에 가장 도움이 된 언어 기능은 **재귀 호출**, **조건식**, 그리고 **함수 정의** 기능이었다.

### 3.1 함수 정의
머꼬에서는 `정의`와 `람다`를 사용하여 함수를 만들 수 있다.  
이 기능 덕분에 계승 계산 과정을 `팩토리얼`이라는 하나의 독립된 함수로 깔끔하게 표현할 수 있었다.  
함수를 따로 정의해 두면 나중에 다른 입력값에 대해서도 같은 코드를 반복해서 사용할 수 있다는 장점이 있다.

### 3.2 조건식
계승은 반드시 `0! = 1`이라는 종료 조건이 필요하다.  
이를 위해 머꼬의 `만약` 구문을 사용하였다.  
이 조건식이 없으면 함수가 계속 자기 자신을 호출하게 되므로, 재귀를 안전하게 끝내기 위해 가장 중요한 기능이라고 생각한다.

### 3.3 재귀 호출
계승 문제를 풀 때 가장 핵심이 되는 기능은 재귀이다.  
`n!`을 계산하기 위해 다시 `(n-1)!`을 구하는 방식은 머꼬와 같은 Lisp/Scheme 계열 언어에서 매우 자연스럽다.  
이번 과제를 통해 재귀는 단순히 “함수가 자기 자신을 다시 부르는 것”이 아니라, **큰 문제를 더 작은 같은 형태의 문제로 바꾸는 방법**이라는 점을 더 분명히 이해할 수 있었다.

## 4. 토의

이번 과제에서 가장 이해하기 쉬웠던 기능은 `만약`을 이용한 조건식이었다.  
입력값이 0인지 확인하고, 맞으면 1을 반환하고 아니면 재귀적으로 계산한다는 흐름은 계승의 정의와 정확히 대응되기 때문에 비교적 쉽게 이해할 수 있었다.  
특히 일반적인 프로그래밍 언어의 `if` 문과 역할이 비슷해서 받아들이기 어렵지 않았다.

반면 가장 처음에 이해하기 어려웠던 부분은 `정의`와 `람다`를 사용하는 함수 표현 방식이었다.  
C나 Python에서는 함수 정의 문법이 비교적 익숙한 편인데, 머꼬에서는 모든 것이 괄호 중심의 리스트 구조로 표현되기 때문에 처음에는 어디까지가 함수 이름이고 어디부터가 인자인지 눈에 잘 들어오지 않았다.  
하지만 여러 번 읽어 보니 머꼬가 Scheme의 키워드를 우리말로 바꾼 형태에 가깝고, 구조 자체는 매우 규칙적이라는 점을 알 수 있었다.

추가로 지원되었으면 하는 기능으로는 **오류 처리** 또는 **입력값 검증** 기능을 들 수 있다.  
예를 들어 음수를 입력했을 때 계승은 일반적인 정의가 어려운데, 현재 프로그램은 그런 경우를 따로 처리하지 않는다.  
만약 머꼬에서 예외 처리나 경고 메시지를 조금 더 편하게 작성할 수 있다면 프로그램을 더 안전하게 만들 수 있을 것이라고 생각한다.

또한 디버깅 과정에서 계산이 어떻게 진행되는지 단계별로 쉽게 볼 수 있는 기능이 더 강화되면 학습용 언어로서 더욱 도움이 될 것 같다.  
특히 재귀 호출은 내부 흐름이 눈에 보이지 않기 때문에, 호출 순서를 시각적으로 보여 주는 기능이 있다면 초보자가 이해하기 훨씬 쉬울 것이다.

## 5. 결론

이번 과제를 통해 머꼬가 단순히 “우리말로 바꾼 특이한 언어”가 아니라, Lisp/Scheme 계열 언어의 핵심 개념을 학습하기 위한 교육용 언어라는 점을 알게 되었다.  
특히 계승 문제를 구현하면서 재귀와 조건식의 관계를 다시 정리할 수 있었고, 함수형 언어에서는 문제의 정의를 코드에 직접 반영하는 방식이 매우 중요하다는 점을 배웠다.

가장 재미있었던 부분은 수학에서 배우는 계승의 정의가 프로그램 코드와 거의 같은 형태로 나타난다는 점이었다.  
반대로 가장 힘들었던 점은 괄호가 많은 문법에 익숙해지는 과정이었다.  
처음에는 괄호의 짝과 함수 구조를 읽는 것이 불편했지만, 익숙해지고 나니 오히려 문법이 매우 단순하고 일관적이라는 장점도 느낄 수 있었다.

종합하면, 이번 과제는 단순히 계승 하나를 구현하는 것 이상의 의미가 있었다.  
재귀적 사고방식, 함수 정의 방식, 조건식의 역할을 직접 코드로 확인할 수 있었고, 이를 통해 Lisp/Scheme 계열 언어의 특징을 조금 더 구체적으로 이해할 수 있었다.
